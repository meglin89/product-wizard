# Hemlock & Oak Product Finder

## Context

Hemlock & Oak has ~290 active products with ~1,900 variants across planners, notebooks, stickers, accessories, and jewelry. The product matrix is complex (paper weight, layout, cover type, size, color) and there's no guided discovery tool — customers must browse collections and understand technical specs on their own. This product finder will guide customers through a step-by-step questionnaire (modeled after Canada's Business Benefits Finder) to narrow down products to their exact needs.

## Scope

**All 8 categories live:** Notebooks, Dated Planners, Undated Planners, Planner Inserts, Stickers, Notepads, Accessories, Jewellery.

## Architecture

**Shopify custom page** at `hemlockandoak.com/pages/product-finder`, with product data provided by Liquid templates (server-rendered JSON). The finder lives inside the store so it inherits the store's header/footer/nav/branding and can link directly to product pages with "View Product" buttons.

**Local dev** uses a static `products.js` (generated by `build_catalog.py`) as a fallback when not running on Shopify.

**Build artifacts:**
```
documents/omicron/shopify product wizard/
  build_catalog.py        # Transforms shopify_products_full.json → products.js
  finder.css              # Mobile-first responsive styles
  finder.js               # Wizard engine, filtering, result rendering, normalization
  products.js             # Static product catalog (generated, local dev fallback)
  index.html              # Local dev shell
```

**Deployment approach (Liquid):**
- Shopify page template outputs raw product JSON via Liquid
- `finder.js` normalizes raw Shopify data client-side (same logic as `build_catalog.py`)
- `products.js` remains as fallback for local development
- CSS/JS uploaded to Shopify's theme Assets folder
- No API tokens, no external requests, no sync jobs needed

**Why Liquid:** Zero credentials exposed, always current (server-rendered on every page load), simplest to implement, most secure.

## Normalized Product Data Model

Each product after normalization:
```javascript
{
  id, title, handle,
  url: "https://hemlockandoak.com/products/{handle}",
  category: "notebooks|dated-planners|undated-planners|inserts|stickers|notepads|accessories|jewellery",
  image: "cdn.shopify.com/...",
  priceMin, priceMax,
  pattern: "dotted|graph|lined|blank",          // notebooks
  paperWeight: "70gsm|120gsm|150gsm",
  coverType: "hardcover|cloth-flex|paper-flex|firm-flex",
  sizes: ["A5", "B5", ...],
  colors: ["Riviere", "Moonflower", ...],
  layout: "weekly|daily|horizontal|minimalist|w&d",  // planners
  guidedJournaling: true|false,
  stickerType: "functional|decorative",          // stickers
  insertType: "weekly|daily|dotted|...",         // inserts
  bindingFormats: ["Ringbound - A5", ...],       // inserts
  inStock: true|false
}
```

**Data quality fixes (in build script and JS normalizer):**
- `Dated Panners` → `dated-planners`
- `Notebook` vs `Notebooks` → `notebooks`
- Empty product_type stickers → classify by tags
- Extract GSM from tags AND title (inconsistent sources)
- Exclude imperfect items, downloads, bundles, hidden, inactive
- Older notebooks (Luminé, Lunar, etc.) default to 150gsm hardcover

## Question Flow / Decision Tree

### Entry: "What are you looking for?"
Large image cards for all 8 categories. Direct categories (notepads, accessories, jewellery) skip the wizard and go straight to results. All-OOS categories (e.g. jewellery) appear greyed out with "Currently sold out" label.

### Flow A: Notebooks
1. **Page pattern** — Dotted / Graph / Lined / Blank / Skip
2. **Paper weight** — 150gsm Heritage / 70gsm Archivist / Skip
3. **Cover style** — Hardcover / Cloth Flex / Paper Flex / Skip
4. **Size** — TN / A5 / B5 / Skip
   - *Auto-skipped if 70gsm selected (A5 only)*
→ Results

### Flow B: Dated Planners
1. **Layout** — Weekly / Horizontal / Daily / Daily Duo / Minimalist Vertical / Minimalist Horizontal / B5 Minimalist / Weekly & Daily / Skip
2. **Cover style** — Hardcover / Cloth Flex / Paper Flex / Skip
→ Results

### Flow C: Undated Planners
1. **Layout** — Weekly (Vertical) / Weekly (Horizontal) / Daily (6-Month) / Weekly & Daily / Skip
2. **Cover style** — Hardcover / Cloth Flex / Paper Flex / Skip
→ Results

### Flow D: Inserts
1. **Binding system** — Ringbound A5 / Discbound Half Letter / Discbound Classic / Discbound A5 / Skip
2. **Insert type** — (checkbox, multi-select) Weekly / Horizontal / Daily / Monthly Calendar / Monthly Review / Dotted / Graph / Blank / Goal Setting / Habit Tracker / Personal Values / Skip
→ Results (OOS products always hidden, no toggle. Shows note about covers + Facebook group link.)

### Flow E: Stickers
1. **Sticker type** — Functional / Decorative / Skip
→ Results

### Flows F-H: Notepads, Accessories, Jewellery
Direct to results (no wizard steps).

## Results Display

**Result cards** (responsive grid: 1-col mobile, 2-col tablet, 3-col desktop):
- Product image
- Title
- Attribute pills (paper weight, cover, layout, sizes, binding formats — each as separate pill)
- Guided Journaling tag (forest green, subtle)
- Price + colour count inline on same line
- View Product button (links to Shopify product page)

**Features:**
- Sort by A-Z / Price Low-High / Price High-Low
- OOS hidden by default with "Show Sold Out (n)" toggle (except inserts — always hidden)
- OOS cards at 55% opacity, sorted to bottom
- Answer summary tags at top of results
- "Edit Answers" and "Start Over" buttons
- Empty state with "Go Back" prompt

## UI/UX Details

- **Mobile-first CSS** — base styles for phones, `min-width` queries scale up
  - 480px+: 2-column category/product grids
  - 700px+: 3-column grids, full desktop spacing
- **Progress bar** with animated dots (scale in/out when steps appear/disappear)
- **Fun facts** auto-rotate every 5 seconds with fade transition (keyed by step ID)
- **Category card hover:** Forest green border + soft shadow + 3px lift (not burgundy)
- **Category images:** `object-fit: contain` on white background
- **URL hash state** for shareability: `#notebooks/pattern=dotted/paperWeight=150gsm`
- **Brand tokens:** Burgundy #7B122E, Forest #1C2914, Bronze #A47738, Cream #F9FAF8
- **No rounded corners** (`--radius: 0px`)

## Special Rules

- **Inserts:** OOS products permanently hidden (no toggle). Permanent note displayed: "We don't currently have our own insert covers available" with link to Facebook community group.
- **Guided Journaling tag:** Shown on dated weekly, horizontal, weekly-and-daily layouts + all undated planners.
- **All-OOS categories:** Greyed out on entry screen (e.g. jewellery).
